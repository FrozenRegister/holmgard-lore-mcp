### Feature — Corpse Ecology (#288)
- Extends the existing `rpg{sub:"corpse"}` handler (previously D&D-style loot only) with Preserve-specific actions: `register` (records a death, snapshotting the deceased's resource-survival inventory via `resource.get_state`), `decompose` (advances through 6 named stages — fresh/early/bloat/active_decay/advanced_decay/skeletal — recomputed fresh from elapsed hours each call), `scavenge_check` (stacked encounter-attraction percentage across corpses at a tile, with a Production-intervention recommendation at 3+ corpses), `loot_corpse` (DEX check against a stage-based DC, contamination filtering, disease exposure roll), `recover` (Production recovery, requires Bloat stage or later), `get_state`, and `psychological_impact` (WIS save against a stage/relationship-based DC with shaken/disturbed/traumatized/break outcome bands).
- `production_intervene`'s `advance_day` now calls the new `tickAllCorpseDecomposition` for every non-recovered corpse in the world — completing the `corpseDecomposition: null` placeholder documented as deferred in #299.
- Corpse loot is tracked as a JSON snapshot on the `corpses` row rather than `corpse_inventory` (FK'd to the D&D `items` table) or `resource_inventory` (its `owner_type` CHECK constraint doesn't include `corpse`, and widening it would require recreating the table under D1) — documented in migration `0014_corpse_ecology.sql`.
- Disease-risk DCs/names and scavenger-attraction percentages per stage are transcribed directly from the issue. Psychological-impact DCs for `early`/`advanced_decay`/`skeletal` (not given by the issue) are interpolated using the same `stageBaseDC + (party_member ? 6 : 0)` formula the issue's own given values satisfy — documented in `corpse-manage.ts`'s file header.
