`world_manage` and `continuity_manage` actions that take an entity/location/faction key now tolerate bare names (e.g. `eira-holt`) by trying common prefixes (`character:`, `npc:`, `location:`, `faction:`, etc.) before failing, and return a `did_you_mean` suggestion when nothing matches. "Invalid params" errors from these tools now state which field is wrong in plain language and include a worked example payload for the action, instead of a bare Zod error dump.
