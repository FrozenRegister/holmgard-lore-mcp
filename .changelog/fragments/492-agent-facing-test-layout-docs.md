### Pointed every agent-facing doc/template at the tests/ layout; auto-enabled the pre-commit hook (#492)

- Audited every doc and template an autonomous agent might read before writing code (`ISSUE_RESOLUTION_PROTOCOL.md`, `CONTRIBUTING.md`, all `.github/ISSUE_TEMPLATE/*.yml`, `.github/PULL_REQUEST_TEMPLATE.md`) — none mentioned that new test files must live under `tests/{unit,worker,live}/` (#488/#489/#490). `ISSUE_RESOLUTION_PROTOCOL.md` Phase 4 in particular — the literal step-by-step an autonomous agent follows — just said "add or update tests" with no path guidance. `CLAUDE.md` was the only place stating the rule, and nothing pointed there.
- Added explicit `tests/unit/` / `tests/worker/` / `tests/live/` guidance (cross-referencing CLAUDE.md as the source of truth) to `ISSUE_RESOLUTION_PROTOCOL.md`, `CONTRIBUTING.md`, `agent_task.yml`, `feature_request.yml`, `refactor.yml`, and the PR template's Test plan section.
- Fixed a second, related staleness: several templates (`bug_report.yml`, `migration.yml`, `feature_request.yml`, `refactor.yml`, the PR template) enumerate the exact `ci.yml` job list and had all drifted out of sync with the `test-layout` job added in #491. Added it to every one.
- New `scripts/setup-git-hooks.mjs`, wired up as the `prepare` npm lifecycle script, so `git config core.hooksPath scripts` runs automatically on `pnpm install` instead of requiring a manual step per clone. Previously, a fresh agent session had zero local pre-commit gate unless someone remembered to run that command by hand — the local `check:test-layout` step (and the rest of the fast gate) simply never fired, and the only feedback was CI, after the commit had already reached GitHub.
