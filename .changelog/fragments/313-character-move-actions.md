### Features
- Added `move_to_location` and `move_to_tile` actions to `character_manage` (#313). `move_to_location` sets `location_key` (a KV lore key); `move_to_tile` sets `current_hex_q`/`current_hex_r`/`map_id`. The two are independent — setting one does not clear the other, matching `spawn-manage.ts`'s `place_character` behavior and letting a character be dual-mode (narrative location + tactical hex position) per the issue's own rule table. The underlying schema (`location_key`, `map_id`, `current_hex_q/r`) already existed via migrations 0024/0026 — this wires up the missing actions.
