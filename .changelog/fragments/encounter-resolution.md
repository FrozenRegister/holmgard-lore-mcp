### Feature — Encounter resolution engine (#280)
- New `rpg({ sub: "encounter", ... })` sub-tool with actions `resolve` (full 1d100 threat check + weighted encounter-type selection + optional injury assignment), `check` (lightweight trigger-only peek, no encounter-type query or injury side effects), `list_types`, `add_type`, and `check_infection`.
- Threat threshold = per-biome `base_threat` (new `biomes.base_threat` column, default 0) + zone threat (from #276's zones, extended with new optional `threatLevel`/`dominanceRank` fields) + contextual modifiers (time of day, noise, scent, party size/injuries, weather — see the issue's modifier table).
- Dominance/suppression: among overlapping zones, the highest-`dominanceRank` zone is "dominant" and contributes full `threatLevel`; every other overlapping zone is a suppressed subordinate contributing at 30% weight. If the selected encounter type belongs to a subordinate zone, the response flags `displaced: true` / `displacedBy: <dominant predator>`.
- Injury severity is derived from how far the roll cleared the threshold (`margin = threshold - roll`) rather than a second independent roll — ties severity organically to how dangerous the triggered encounter was. Injuries persist to a new `character_injuries` table when `characterIds` are supplied (unpersisted, response-only, when resolving for a generic party).
- `check_infection` reports fever/sepsis staging from a persisted injury's severity + elapsed time + treatment received.
- `travel_manage`'s `travel` action gains an optional `resolveEncounter` param that calls the same engine when the caller also supplies `worldId`/`x`/`y` (room_nodes, travel's own location model, has no world/coordinate linkage at all); otherwise it falls back to the pre-existing flat 15% flag, unchanged.
- `encounter_types` has no auto-seeded defaults (unlike biomes) — predator rosters are narrative-specific per world, so every world's roster is narrator-authored via `add_type`.

Design decisions made explicitly, not fully specified by the issue (see `src/rpg/handlers/encounter-manage.ts`'s file header for the full rationale): the issue assumed `query_zone` (#276) already returned "resolved threat data with dominance suppression," but #276 as shipped carried only shape/type/predator — no threat/dominance concept existed until this PR added it.
