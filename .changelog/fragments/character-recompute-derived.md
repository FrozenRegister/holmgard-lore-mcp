### Feature — `character.recompute_derived` action (#266, #269)
- Adds `recompute_derived` to `character_manage`: recalculates `ac`/`perception_bonus`/`stealth_bonus` from a character's own already-stored `stats` (`10+DEXmod`, `WISmod`, `DEXmod` — the exact formula `get`'s display-only fallback already uses) and persists the result. Accepts a single `id`/`characterId`, or a `worldId` to bulk-recompute every character in that world.
- Investigation into #266 (`character.update` resetting level/AC/perception/stealth) and #269 (`born` null on all characters) found both underlying code bugs already fixed — `update` already has correct PATCH semantics, `create`/`update` already accept `born`, and live production data confirms `level`/`born` are already correctly populated for every Calder-world character. The one remaining, purely mechanical gap: `ac`/`perception_bonus`/`stealth_bonus` were never re-derived after the level-restoration repair, so they're stale relative to each character's actual ability scores. `recompute_derived` closes that gap without any narrative-data guessing — everything it touches is already in the database.
- `character.list` already includes `born` on every row (added by #305) — no change needed there.
