`world_manage` and `continuity_manage` now accept the most commonly-guessed alternate parameter names as aliases (`entity_name`→`entity_key`, `faction_name`→`faction_key`, `location_id`→`location_key`, `setup_id`→`id`, `goal_name`→`goal_id`, `goal_description`→`description`, `date`→`at`, `description`→`detail` for `append_event`, and `low`/`medium`/`moderate`/`high`/`critical` as aliases for `check_continuity`'s `severity_floor`). Aliases are declared directly in each tool's JSON Schema (not just accepted server-side), so MCP clients that validate `arguments` against the schema before sending a request (e.g. Cline) will also accept them.
