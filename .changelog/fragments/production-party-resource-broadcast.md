### Feature — Production Cycle, Party Trust & Betrayal, Resource Survival, Broadcast layer (#283, #285, #286, #287)

Shipped together in one PR — the four systems are genuinely circularly coupled by content (production's `advance_day` drives resource degradation/crate drops and reads broadcast state; broadcast votes pressure party betrayal; party's shared resources are tracked by the resource system), not merely by the issues' own (incorrectly cross-referenced) citation numbers.

- **`rpg{sub:"production"}`** (new) — `advance_day` (perimeter contraction, hazard escalation, weather roll, prize drops, audience-vote scheduling, extraction window), `get_state`, `set_schedule`, `list_events`. New `production_calendar` table; `world_state` gains `production_day`/`perimeter_radius`/`weather`/`hazard_level`/`encounter_modifier`/`extraction_window`/`last_intervention_at`/`production_mood`; `characters` gains a `production_state` JSON column.
- **`rpg{sub:"party"}`** (extended, not duplicated — see below) gains `trust_shift`, `resolve_conflict`, `betrayal_check`, `morale_roll`, `watch_rotation`. `parties` gains `morale`/`cohesion`/`watch_order`/`current_watch`; new `party_trust` pairwise matrix table. `form`/`get_state` alias to the existing `create`/`get`.
- **`rpg{sub:"resource"}`** (new) — `crate_drop`, `consume`, `degrade`, `improvise`, `scavenge`, `craft`, `get_state`. New `resource_inventory`, `resource_owner_state`, `crate_drops` tables.
- **`rpg{sub:"broadcast"}`** (new) — `audience_pulse`/`trigger_event` (aliased, same logic), `resolve_vote`, `production_intervene`, `celeste_moment`, `get_state`. New `broadcast_approval`, `broadcast_votes`, `broadcast_interventions` tables.

**Architectural note:** #285 assumed `rpg{sub:"party"}` didn't exist yet, but it already did (generic create/get/list/add_member grouping used by combat and world-manage). Rather than adding a second, colliding "party" concept, this PR extends the existing handler/table — see migration `0013_production_party_resource_broadcast.sql` and `docs/storage-selection-kv-vs-d1.md` context for the reasoning.

**Explicit scope decisions** (documented in each handler's file header, not guessed silently): several mechanics the issues describe but don't fully specify (resolve_conflict's outcome bands, forage/craft DCs not given in the issue, production_intervene's signal inputs, betrayal_check's desperation multipliers) are this PR's own synthesis. Destructive/irreversible consequences (mercy-kill execution, forced party dissolution, resource splitting) are never auto-applied — handlers return the mechanical decision for the caller to act on. Corpse decomposition (#288, not yet shipped) is a documented no-op in `advance_day`, not a silent gap.
