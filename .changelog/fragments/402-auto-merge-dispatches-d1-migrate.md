### CI — fix D1 migrations silently never reaching production after auto-merge
- `auto-merge.yml`'s squash-merges (via `GITHUB_TOKEN`) don't trigger other workflows' `push`/`pull_request` listeners — the same GitHub anti-recursion behavior already documented here for closing-keyword automation. `d1-migrate.yml` stopped firing once auto-merge became the primary merge path (2026-07-11), leaving production 8 migrations behind (`0024`-`0031`) before being caught manually. See #402.
- `auto-merge.yml` now checks whether a merged PR touched `schema/migrations/**` and, if so, directly dispatches `d1-migrate.yml` via `workflow_dispatch` — the same pattern `ci.yml`/`pr-quality.yml` already use to dispatch `auto-merge.yml` itself. Best-effort; a dispatch failure never fails the merge job.
