### Fixed

- **MCP transport bridge stringification (#505)**: Some MCP client transports (e.g. the Shapes bridge) `JSON.stringify()` every non-string `tools/call` argument before it reaches this Worker, so booleans, numbers, arrays, and objects arrived as their stringified text (`true` → `"true"`, `0.7` → `"0.7"`, an array → its JSON text) and failed zod validation in every handler (`Expected boolean, received string`, etc.). Added `coerceTransportArgs()` (`src/lib/coerce-transport-args.ts`), applied to `tools/call` arguments at both transport entry points (`src/index.ts`'s JSON-RPC handler and `src/do/HolmgardMCP.ts`'s Streamable HTTP DO handler) before dispatch. Any string that round-trips through `JSON.parse()` into a boolean, number, array, or object is replaced with the parsed value; ordinary prose strings are never valid JSON on their own and pass through unchanged.
