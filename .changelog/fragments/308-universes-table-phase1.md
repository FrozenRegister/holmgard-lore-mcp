### Features
- Added `universes` table (Phase 1 of #308): a narrative-namespace parent that will eventually let a story span multiple `worlds` (spatial grids). This PR is purely additive/non-breaking — a new `universes` table plus nullable `universe_id` columns on `worlds`, `timeline_events`, `production_calendar`, `resource_inventory`, `crate_drops`, `broadcast_votes`, and `broadcast_interventions`. No backfill, no renames, no `NOT NULL` constraints, and no code paths read or write `universe_id` yet. Phase 2 (nullable `width`/`height` on `worlds`) and Phase 3 (the `worlds` → `maps` rename plus cross-world query support) remain open and undecided — see the Phase 1 gameplan comment on #308.
