### Fix — `/admin/map/push-hexes`/`push-landmarks` no longer wipe RPG-owned columns (#321)
- Both routes used `INSERT OR REPLACE`, which reset every column not in the statement (all RPG-owned columns added by migration `0019`/#319 — `world_id`, `biome`, `region_id`, `population`, `zone_type`, `zone_shape`, `predator_ref`, `threat_level`, `dominance_rank`) to their schema defaults on every ordinary editor push. Switched both to `INSERT ... ON CONFLICT DO UPDATE SET <only the columns each route owns>`, preserving any RPG-set value the editor's push doesn't carry. See `docs/issues/HIGH-map-push-insert-or-replace-wipes-rpg-columns.md`.
- `push-hexes` now also accepts optional `worldId`/`biome` per hex (used by `holmgard-lore-editor`'s new biome-assignment panel), preserving the existing value via `COALESCE` when omitted.
- `/internal/map-readback` now additionally returns `worldId`/`biome` (null when unset) on each hex, so the editor can read back a hex's current biome assignment without a separate round-trip.
