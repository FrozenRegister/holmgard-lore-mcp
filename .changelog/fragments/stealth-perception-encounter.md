### Feature — Stealth/perception opposed checks integrated with the encounter engine (#284)
- `perception_manage` gains `stealth_check` (Yield stealth vs. predator perception, 1d20 + bonus + situational modifiers each side) and `perception_contested` (generic opposed check between any two observers/actors). Outcome bands: `avoided_entirely` (margin >= 5), `tense_moment` (1-4), `predator_searching` (tie), `yield_spotted` (-1 to -4, Yield advantage), `ambushed` (<= -5, predator advantage).
- Modifier tables (transcribed from the issue): predator perception — distance zone core +5/edge -3, wind toward +4/away -4/crosswind +1, Yield bleeding +6, cooking/fire +3. Yield stealth — mode hiding +2/active +0/passive -5/rushed -8, cover type forest-like +3/open-like -3/wet-like +1, night +2, party size -2 per member beyond 1.
- `encounter_manage`'s `resolve`/`check` gain an optional `stealthCheck` param that runs the opposed check first: `avoided_entirely`/`tense_moment` short-circuits with `confrontationAvoided: true` and skips the threat roll entirely; any other outcome continues the normal threshold pipeline with `stealthResult` attached to the response.
- `predatorPerceptionModifier`, `yieldStealthModifier`, and `stealthOutcomeFromMargin` are exported from `perception-manage.ts` so `encounter-manage.ts` reuses the same math rather than duplicating it.
