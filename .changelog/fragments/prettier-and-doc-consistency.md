### Added Prettier for code formatting, auto-fix CI, and cross-doc consistency fixes

- Introduced Prettier for `.ts`/`.mjs` files (`pnpm run format` / `format:check`), configured to match the codebase's existing style (no semicolons, single quotes, 2-space indent, 100-char width) rather than Prettier's defaults, to keep the one-time reformat minimal. Paired with `eslint-config-prettier` so ESLint and Prettier never fight over the same rule.
- New `Auto-fix Code Formatting` CI workflow mirrors the existing `Auto-fix Markdown` pattern: runs `pnpm run format` and pushes any fix directly back to the PR branch.
- Applied the one-time repo-wide reformat (271 files). Purely mechanical — verified behavior-safe via the full test suite (129 files / 2910 tests) and unit tier (10 files / 153 tests), both matching pre-reformat counts exactly. One real bug fixed along the way: Prettier expanded a single-line function signature in `src/rpg/handlers/tick-hooks.ts`, breaking 5 `eslint-disable-next-line` comments that no longer landed on the right line — moved to trailing `eslint-disable-line` comments that travel with the code.
- Every agent-facing doc/template (`ISSUE_RESOLUTION_PROTOCOL.md`, `CONTRIBUTING.md`, all issue templates, the PR template) now explicitly states new test files go under `tests/unit/`, `tests/worker/`, or `tests/live/` — previously only `CLAUDE.md` said so.
- The local pre-commit gate now self-activates: `scripts/setup-git-hooks.mjs` runs `git config core.hooksPath scripts` automatically via the `prepare` npm lifecycle script. Also fixed a real bug found while verifying this: `scripts/pre-commit` (the hook entry point) was committed without the executable bit, so git was silently skipping it for every clone regardless of `core.hooksPath` configuration.
