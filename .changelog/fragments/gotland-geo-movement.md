### Feature — Real-world-distance-constrained party movement for Gotland (#328)

- Adds a per-world `waypoint` sub-tool (`register`/`list`/`get`/`update`/`delete`/`validate`/`seed_defaults`/`calibrate`/`hex_to_latlon`): named, real-lat/lon-anchored locations plus a precomputed pairwise foot-routing distance table. Mechanism is reusable by any world; the actual Gotland place set (Visby, Roma Kloster, Fårösund, Klintehamn) is only seeded into a world that opts in via `waypoint.seed_defaults` — not auto-seeded on `world.create`, since real Swedish place names only make sense for a Gotland-set campaign.
- Adds `src/rpg/utils/geo-transform.ts`: a lat/lon &lt;-&gt; hex-axial `(q, r)` placement helper reusing `world-map.ts`'s existing `hexToPixel` formula (kilometers standing in for pixels) with a single origin + scale, rather than a general N-point affine fit.
- Adds `scripts/gotland-precompute-distances.mjs`: an offline, one-time script that calls the free OSRM foot-routing demo instance for every waypoint pair and writes the static seed data (`schema/seed-data/gotland-waypoints.json`, `gotland-distance-matrix.json`). The Worker never makes a live routing call at request time — an unrouted pair's `distance_km` is NULL, surfaced as a structured `{blocked: true, reason}` response, never a tool error.
- Adds `party_manage` actions `begin_march`/`get_march_status` and an exported `tickAllPartiesMarch(db, worldId, daysToAdvance)` helper. Movement resolves per-party via an explicit call — deliberately **not** wired into `production_manage`'s `advance_day` (which ticks the whole world's hazard/weather/resource-degradation at once), so one party's march never blocks on another party's turn.
- `travel_pace_km_per_day` defaults to 24 km/day via the migration's D1 column `DEFAULT` — this is the only place that number lives; application code never hardcodes a fallback pace.
- Migration `0021_gotland_waypoints_and_party_march.sql`: adds `world_state.geo_origin_lat/lon`/`geo_km_per_hex`, new `waypoints`/`waypoint_distances` tables, and 7 new movement columns on `parties` (table-rebuild pattern, matching migration 0019's precedent).
- Discovery: OSRM's foot-routing profile finds a route across every pair in the seeded 4-waypoint Gotland set, including the Fårösund strait (apparently via the free ferry) — the originally-assumed "no route" demo case doesn't occur in the shipped seed data; the blocked-response path is covered by unit tests against synthetic fixtures instead. See `docs/holmgard-user-guide.md`.
